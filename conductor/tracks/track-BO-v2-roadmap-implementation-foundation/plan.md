# Track BO: v2 Roadmap Implementation Foundation Plan

## Phase BO1 - Stable-before-v2 gate

- [ ] Task: Define v2 eligibility.
    - [ ] Require stable reader baseline evidence.
    - [ ] Require at least one real public-quality installable artifact.
    - [ ] Require marketing claim governance and parity registry checks to pass.
- [ ] Task: Conductor phase gate.
    - [ ] Run `scripts/conductor_phase_gate.sh --phase BO1 --auto-fix`.

## Phase BO2 - v2 roadmap design

- [ ] Task: Create v2 roadmap.
    - [ ] Sequence advanced parity, workflow packs, source-linked authoring, collaboration, marketplace, intelligence, enterprise and mobile maturity.
    - [ ] Define feature gates, rollback, owners and evidence.
    - [ ] Split implementation tranches into small tracks.
- [ ] Task: Conductor phase gate.
    - [ ] Run `scripts/conductor_phase_gate.sh --phase BO2 --auto-fix`.

## Phase BO3 - First v2 implementation tranche

- [ ] Task: Implement first post-stable v2 tranche.
    - [ ] Choose the highest-value stable-adjacent feature family.
    - [ ] Add contracts, fixtures, UI/CLI flows, tests and docs.
    - [ ] Attach evidence to quality dashboard and release notes.
- [ ] Task: Conductor phase gate.
    - [ ] Run `scripts/conductor_phase_gate.sh --phase BO3 --auto-fix`.

## Phase BO4 - v2 quality and adoption loop

- [ ] Task: Establish v2 release cadence.
    - [ ] Maintain stable bug/security response gates.
    - [ ] Publish evidence-backed roadmap status.
    - [ ] Keep unsupported v2 claims out of marketing copy.
- [ ] Task: Conductor phase gate.
    - [ ] Run `scripts/conductor_phase_gate.sh --phase BO4 --auto-fix`.
