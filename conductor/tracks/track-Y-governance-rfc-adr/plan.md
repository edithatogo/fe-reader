# Track Y: Maintainer Governance, RFCs & ADRs

## Mission

Keep the project coherent as it grows across PDF engine, UI, platforms, workflows, SDKs and integrations.

## Phases

### Y0 ADR schema and seed ADRs

Deliver ADR schema, validation script and seed ADR list.

### Y1 RFC triggers

Define which changes require RFCs and which can be normal pull requests.

### Y2 Maintainer domains

Define review owners for core PDF, security/redaction, platform integration, performance, accessibility/i18n, release and workflow packs.

### Y3 Contribution quality gates

Define issue templates, review templates and corpus-fixture requirements.
