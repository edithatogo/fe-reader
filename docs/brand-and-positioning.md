# Brand and Positioning

This file exists as a compatibility alias for older checks and references that expect `docs/brand-and-positioning.md`.

The canonical guardrail document is `docs/ip-brand-originality-guardrails.md`. Keep that file as the source of truth for product language, originality, license review, and public-positioning review.

## Positioning

Fe Reader is a local-first PDF workflow platform. It should be described in neutral capability terms:

- verified document operations
- workflow packs
- document health
- safe automation
- metadata transparency
- privacy-preserving diagnostics

## Guardrails

- Avoid vendor-clone language in public docs.
- Avoid UI or interaction copying that imitates a proprietary layout.
- Use standards-based feature names instead of branded names.
- Keep an ADR for any feature inspired by a commercial baseline.

## Review gate

Before public beta, review:

- trademark and naming risk
- icon and UI originality
- dependency license hygiene
- patent-risk triage for unusual workflows
- export-control or crypto distribution concerns where applicable
